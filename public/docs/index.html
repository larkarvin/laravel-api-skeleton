Introduction

This is a test API for Arvin Almario.

This documentation aims to provide all the information you need to work with our Task API.

Installation

composer install

Running the Server

php artisan serve

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Base URL

http://localhost

Authenticating requests

Authenticate requests to this API's endpoints by sending an Authorization header with the value "Bearer {YOUR_AUTH_KEY}".

All authenticated endpoints are marked with a requires authentication badge in the documentation below.

You can retrieve your token by using the Login and Request a token API Endpoint .

Endpoints

Register a New User.

Example request:
$client = new \GuzzleHttp\Client();
$response = $client->post(
    'http://localhost/api/v1/register',
    [
        'headers' => [
            'Content-Type' => 'application/json',
            'Accept' => 'application/json',
        ],
        'json' => [
            'email' => 'arvin@almar.io',
            'name' => 'Arvin Almario',
            'password' => 'ultimatepassword',
        ],
    ]
);
$body = $response->getBody();
print_r(json_decode((string) $body));
curl --request POST \
    "http://localhost/api/v1/register" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"email\": \"arvin@almar.io\",
    \"name\": \"Arvin Almario\",
    \"password\": \"ultimatepassword\"
}"

Example response (200):

    "token": "eyJ0eXA..."
}

Body Parameters

Must be a valid date in the format Y-m-d. This field is required when due_date_to is present. Must be a date before due_date_to.

Must be a valid date in the format Y-m-d. This field is required when due_date_from is present.

completed_date_from  string optional  

This field is required when completed_date_to is present. Must be a valid date in the format Y-m-d. Must be a date before completed_date_to.

created_date_to  string optional  

Must be a valid date in the format Y-m-d.

Must be a valid date in the format Y-m-d. This field is required when archived_date_to is present. Must be a date before archived_date_to.

archived_date_to  string optional  

Must be a valid date in the format Y-m-d. This field is required when archived_date_from is present.

sort_by  string optional  

Must be one of title, description, due_date, created_at, completed_date, or priority.

order_by  string optional  

Must be one of desc, asc, DESC, or ASC.

priority  string optional
